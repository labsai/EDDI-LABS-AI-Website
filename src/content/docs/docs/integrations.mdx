EDDI is built on proven open-source foundations — LangChain4j, Quarkus, and the Model Context Protocol — making it compatible with a wide ecosystem of AI tools and infrastructure.

## LLM Providers

EDDI supports **20+ LLM providers** through LangChain4j. Switch between providers by changing a JSON configuration — no code changes required.

<CardGrid>
  <Card title="OpenAI" icon="rocket">GPT-4o, GPT-4o-mini, o1, and all OpenAI models. Full tool calling support.</Card>
  <Card title="Anthropic" icon="puzzle">Claude Sonnet, Claude Opus, Claude Haiku. Native function calling.</Card>
  <Card title="Google Gemini" icon="star">Gemini Pro, Gemini Flash, and the full Vertex AI family.</Card>
  <Card title="Azure OpenAI" icon="setting">Enterprise Azure deployments with managed endpoints and private networking.</Card>
  <Card title="Ollama" icon="laptop">Run local models entirely on-premises. Zero data leaves your network.</Card>
  <Card title="Amazon Bedrock" icon="approve-check-circle">Access Claude, Llama, Titan, and other models via AWS infrastructure.</Card>
  <Card title="Mistral" icon="rocket">Mistral Large, Medium, and Small. European-hosted LLM option.</Card>
  <Card title="And More..." icon="list-format">Cohere, Together AI, Fireworks, Groq, DeepSeek, and additional providers via LangChain4j.</Card>
</CardGrid>

## MCP Ecosystem

<CardGrid>
  <Card title="MCP Server" icon="puzzle">
    EDDI exposes **39 tools** via MCP — compatible with Claude Desktop, Cursor, Windsurf, Continue, and any MCP-compatible client.
  </Card>
  <Card title="MCP Client" icon="external">
    Agents can **consume external MCP tools** from other servers during conversations via StreamableHTTP transport. Extend agent capabilities without code changes.
  </Card>
</CardGrid>

## Authentication & Security

<CardGrid>
  <Card title="Keycloak / OIDC" icon="approve-check-circle">
    Enterprise authentication with native Keycloak integration. Complex OAuth2/OIDC topologies, role-based access control, and SSO.
  </Card>
  <Card title="Vault Integration" icon="setting">
    API keys and secrets managed through Quarkus Vault/Secrets Manager. Referenced via secure `${vault:key}` URIs — never stored in agent configurations.
  </Card>
</CardGrid>

## Deployment

<CardGrid>
  <Card title="Docker" icon="setting">
