EDDI is an **enterprise-grade AI orchestration middleware** built on Java 25 and Quarkus. It provides a configuration-driven platform for deploying, managing, and scaling AI agents and multi-agent systems.

## Prerequisites

- **Docker** (recommended) or Java 25+
- An LLM provider API key (OpenAI, Anthropic, Google Gemini, or a local Ollama instance)

## Quick Start with Docker

The fastest way to run EDDI is with Docker:

```bash
docker run -p 7070:7070 labsai/eddi:latest
```

EDDI will be available at `http://localhost:7070`.

## Connect via MCP

If you use **Claude Desktop** or any MCP-compatible client, add EDDI as an MCP server:

```json
{
  "mcpServers": {
    "eddi": {
      "url": "http://localhost:7070/mcp"
    }
  }
}
```

Now you can interact with EDDI's 39 MCP tools directly from your AI assistant.

## Create Your First Agent

Use the `setup_agent` MCP tool to create a fully working agent in one call:

```
setup_agent(
  name: "My first agent",
  systemPrompt: "You are a helpful assistant that answers questions clearly.",
  provider: "anthropic",
  modelId: "claude-sonnet-4-6"
)
```

This creates the rules, LLM config, workflow, agent, and deploys it — all in one step.

## Chat with your agent

```
chat_with_agent(agentId: "<your-agent-id>", message: "Hello! What can you do?")
```
