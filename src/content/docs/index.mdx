    <h2 class="text-3xl font-bold text-white! border-0! text-center">Frequently Asked Questions</h2>
  </div>

<details>
<summary><strong>What is EDDI?</strong></summary>

EDDI is an open-source, JVM-native visual AI orchestration middleware built on Quarkus and Java 25. It enables enterprise teams to build, configure, and deploy AI-powered agents using JSON configuration rather than compiled code. EDDI provides a complete platform with built-in REST APIs, conversation state management, security (OIDC/Keycloak), audit trails, and 39 MCP tools — all deployable via Docker or Kubernetes.

</details>

<details>
<summary><strong>How is EDDI different from Spring AI or LangChain4j?</strong></summary>

Spring AI and LangChain4j are excellent **libraries** — EDDI actually uses LangChain4j internally. The key difference is that EDDI is a **deployable middleware platform**, not a library. With Spring AI, you still need to build REST controllers, authentication, conversation state management, audit logging, and management UIs yourself. EDDI provides all of this out of the box, enabling prompt engineers to iterate on agent rules instantly via UI or API without touching Java code.

</details>

<details>
<summary><strong>Is EDDI production-ready for enterprise use?</strong></summary>

Yes. EDDI runs on Java 25 with Quarkus, leveraging Virtual Threads for massive I/O-bound concurrency. It supports MongoDB and PostgreSQL, includes built-in OIDC/Keycloak authentication, provides immutable cryptographic audit trails for compliance (including EU AI Act), and scales horizontally via NATS JetStream. It has no `eval()` or code execution blocks, eliminating an entire class of security vulnerabilities.

</details>

<details>
<summary><strong>What is the Model Context Protocol (MCP) and how does EDDI use it?</strong></summary>

The Model Context Protocol (MCP) is an open standard that allows AI assistants like Claude Desktop, IDE plugins, and custom clients to interact with external tools programmatically. EDDI exposes 39 MCP tools spanning conversation management, agent administration, setup automation, schedule management, and diagnostics. Agents can also act as **MCP clients**, consuming tools from external MCP servers during conversations.

</details>

<details>
<summary><strong>Can EDDI replace Flowise or n8n for production workloads?</strong></summary>

EDDI serves a similar visual-building purpose but on the JVM instead of Node.js. Unlike Flowise and n8n, EDDI uses no `eval()` or code blocks (eliminating CVSS 10.0 risks), runs Java 25 Virtual Threads for enterprise-grade concurrency, supports OIDC/Keycloak authentication, and stores data in MongoDB or PostgreSQL rather than SQLite. It is designed for regulated industries where security, auditability, and JVM-native performance are requirements.

</details>
