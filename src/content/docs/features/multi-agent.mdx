## Multi-Agent Capabilities

<CardGrid stagger>
  <Card title="Agent Discovery" icon="magnifier">
    `discover_agents` returns deployed agents **enriched with intent mappings** from agent triggers. Find the right agent by purpose, not by ID.
  </Card>
  <Card title="Managed Conversations" icon="comment-alt">
    `chat_managed` creates **one conversation per intent+userId**, auto-created on first message. No conversation lifecycle management in your app code.
  </Card>
  <Card title="Agent Triggers" icon="setting">
    Map intents to agent deployments. Route `customer_support` to Agent A in production, Agent B in staging. **A/B test** with weighted routing.
  </Card>
  <Card title="Environment Isolation" icon="approve-check-circle">
    Deploy the same agent to `test`, `restricted`, and `unrestricted` environments. Each environment has its own conversation space.
  </Card>
</CardGrid>

## How It Works

```
1. Create a trigger: intent="billing" → agentId="billing-agent"
2. User sends: chat_managed(intent="billing", userId="user1", message="What's my balance?")
3. EDDI:
   a. Looks up trigger for "billing" intent
   b. Finds or creates a conversation for userId="user1" + intent="billing"
   c. Routes the message to the billing-agent
   d. Returns the response
4. Next message from same user + intent reuses the same conversation
```
