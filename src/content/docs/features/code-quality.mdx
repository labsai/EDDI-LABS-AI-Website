## Frequently Asked Questions

<details>
<summary><strong>How many tests does EDDI have?</strong></summary>

EDDI has **over 1,000 tests with zero failures** in its main repository, plus additional end-to-end tests in the dedicated [EDDI-integration-tests](https://github.com/labsai/EDDI-integration-tests) repository. This comprehensive coverage means you can refactor, extend, and contribute with confidence — the test suite catches regressions before they reach production.

</details>

<details>
<summary><strong>What test framework does EDDI use?</strong></summary>

EDDI uses **JUnit 5** with **Quarkus Test** extensions. Tests run in a real Quarkus container with CDI injection, making them true integration tests rather than superficial mocks. Mockito is used selectively for external service boundaries (LLM providers, databases in unit tests).

</details>

<details>
<summary><strong>How do I run the test suite locally?</strong></summary>

Run `mvn test` for unit tests or `mvn verify` for the full integration test suite. Quarkus Dev Services automatically spins up MongoDB and PostgreSQL containers — no manual database setup required. For the full end-to-end suite, clone the [EDDI-integration-tests](https://github.com/labsai/EDDI-integration-tests) repo and run it against a running EDDI instance.

</details>

<details>
<summary><strong>Does EDDI follow any specific coding standards?</strong></summary>

EDDI follows standard Java conventions with Quarkus idioms: CDI injection over static factories, reactive patterns for I/O, immutable DTOs, and builder patterns for complex objects. The codebase is organized by domain (agent management, conversations, MCP, security) rather than by technical layer, making it intuitive for contributors to navigate.

</details>
