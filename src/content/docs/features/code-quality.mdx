    <h2 class="text-3xl font-bold text-white! border-0! text-center">Frequently Asked Questions</h2>
  </div>

<details>
<summary><strong>How many tests does EDDI have?</strong></summary>

EDDI has **over 1,000 tests with zero failures** in its main repository, plus additional end-to-end tests in the dedicated [EDDI-integration-tests](https://github.com/labsai/EDDI-integration-tests) repository.

</details>

<details>
<summary><strong>What test framework does EDDI use?</strong></summary>

EDDI uses **JUnit 5** with **Quarkus Test** extensions. Tests run in a real Quarkus container with CDI injection, making them true integration tests.

</details>

<details>
<summary><strong>How do I run the test suite locally?</strong></summary>

Run `mvn test` for unit tests or `mvn verify` for the full integration test suite. Quarkus Dev Services automatically spins up MongoDB and PostgreSQL containers.

</details>

<details>
<summary><strong>Does EDDI follow any specific coding standards?</strong></summary>

EDDI follows standard Java conventions with Quarkus idioms: CDI injection over static factories, reactive patterns for I/O, immutable DTOs, and builder patterns for complex objects.

</details>
</section>
