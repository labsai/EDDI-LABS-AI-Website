  </div>

<details>
<summary><strong>Can an AI coding agent modify EDDI's agent configurations?</strong></summary>

Yes — this is EDDI's primary design goal. Agent logic (prompts, pipelines, tool definitions) is **JSON configuration**, not compiled Java code. AI agents can read and modify configurations via the REST API or MCP tools without any Java knowledge.

</details>

<details>
<summary><strong>How does EDDI's MCP server help coding agents?</strong></summary>

EDDI's MCP server exposes 39 tools that cover the full agent lifecycle: creation, configuration, deployment, conversation, scheduling, and diagnostics. The MCP tool schemas include detailed parameter descriptions, making them self-documenting.

</details>

<details>
<summary><strong>Is the EDDI codebase easy for AI tools like Copilot or Cursor to understand?</strong></summary>

Yes. EDDI's codebase is designed with AI-readability in mind: consistent naming conventions, small focused classes (under 200 lines average), typed configuration models, CDI dependency injection, and **1,000+ tests with zero failures** ensuring safe refactoring.

</details>
</section>
