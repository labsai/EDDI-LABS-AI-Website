## Debugging AI is Hard. EDDI Makes It Visible.

When an AI agent produces an unexpected response, you need to answer:

1. **What prompt** did the LLM actually see?
2. **What tools** were called, and what did they return?
3. **Which rule** fired, and why?
4. **How much** did this conversation cost?

EDDI's audit trail answers all four questions for every single turn — via the `read_audit_trail` and `read_conversation_log` MCP tools, or via the REST API.

## EU AI Act Ready

The EU AI Act requires "high-risk AI systems" to maintain comprehensive logging. EDDI's audit ledger provides:

- **Traceability** — every AI decision is logged with its inputs and outputs
- **Accountability** — which model, which version, which configuration produced this response
- **Reproducibility** — full conversation context preserved for review
